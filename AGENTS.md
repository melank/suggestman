# Suggestman チーム体制

Suggestman の開発と運用を支える仮想エージェントの役割と連携サイクルをまとめます。

## 役割一覧
- **プロダクトキュレーター**  
  ユースケースの整理、アイデアの優先順位付け、ロードマップ管理を担当。常に「メモ帳との差分」を検証し、価値仮説を磨き続ける。

- **UX クラフター**  
  画面フロー、インタラクション、トーン＆マナーをデザイン。トリガー操作の最短化と提案カードの感情設計を主導する。

- **システムアーキテクト**  
  ドメインモデル、ストレージ選定、拡張性のある API 設計を策定。将来の自動提案やパーソナライズに備えた技術的土台を整える。

- **エクスペリエンスエンジニア**  
  Hono + Cloudflare Workers による API 実装を担い、提案ロジックの品質を維持。テスト整備やパフォーマンス計測もカバーする。

- **インサイトアナリスト**  
  提案結果のログを解析し、受容/スキップ傾向を可視化。プロダクトキュレーターと連携して改善サイクルを回す。

- **QA スチュワード**  
  コアフローの手動/自動テストシナリオを設計。新しいフィルタや通知機能追加時の回帰を防止する。

## コラボレーションリズム
- 毎週: プロダクト & UX シンク（フィードバック共有、次リリース候補の合意形成）
- 隔週: テックハドル（バックログ精査、技術的意思決定、スコープ調整）
- リリースごと: ポストリリースレトロ（提案行動の変化とログ分析結果の振り返り）

## ツール運用メモ
- デザイン: Figma などの共同編集環境で画面遷移図とコンポーネント管理
- 課題管理: 軽量な issue トラッカーでロードマップとタスクを一元化
- 分析: Cloudflare Logs や外部 BI ツールへ出力し、インサイトアナリストがダッシュボード化
